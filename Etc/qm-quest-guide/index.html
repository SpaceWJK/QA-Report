QA 업무를
퀘스트처럼 관리하기

INTRODUCTION

QM-Quest란? Core Concepts

처음 보면 낯선 용어 4개만 알면, 나머지는 자연스럽게 이해돼요.

No. 01
퀘스트
Quest
우리가 흔히 말하는 "업무" 한 건입니다. 제목·마감일·담당자(파티장)를 가지고, 진행률이 0%에서 100%까지 올라갑니다.
No. 02
에피소드
Episode
여러 퀘스트를 묶는 상위 단위입니다. 프로젝트나 캠페인처럼, 관련된 업무들을 하나로 모아 진행 상황을 함께 봅니다.
No. 03
길드
Guild
회사 조직·팀 단위가 아닙니다. 개인의 성장을 목표로, 공통 관심 주제에 따라 자발적으로 모인 파티예요. 멤버가 퀘스트를 완료하면 포인트가 쌓여 길드 등급이 올라가고, 다른 길드와 랭킹으로 경쟁합니다.
No. 04
파티장
Leader
퀘스트를 실제로 책임지는 사람입니다. 진행률 업데이트, 완료 요청 등 퀘스트의 핵심 조작은 파티장(또는 관리자·매니저)만 할 수 있어요.
비유로 정리하면 — "퀘스트=내가 해야 할 업무", "에피소드=업무들을 묶은 프로젝트", "길드=같은 관심사로 모여 함께 성장하는 스터디 파티"라고 생각하시면 됩니다. 업무를 완료할 때마다 포인트를 얻고, 등급이 오르는 게임 같은 경험을 QA 업무에 그대로 적용한 서비스예요.

GETTING STARTED
04 / 20
시작하기 Login & Approval
Step 01
로그인
이름과 비밀번호로 로그인합니다. 계정이 없다면 가입 신청을 먼저 진행하세요.
Step 02
승인 대기
가입 신청 후에는 관리자의 승인이 필요합니다. 승인 전까지는 로그인이 제한될 수 있어요 — 조급해하지 않으셔도 됩니다.
Step 03
이용 시작
관리자가 승인하면 바로 이용 가능합니다. 이때 역할(관리자/매니저/멤버/뷰어)이 함께 부여됩니다.
가입 신청→ 관리자 승인 대기→ 승인 완료 · 역할 부여→ 로그인 & 이용
내 역할이 궁금하다면 — 우측 상단 프로필 또는 설정 메뉴에서 확인할 수 있어요. 역할에 따라 보이는 메뉴와 할 수 있는 조작이 달라지니, 헷갈리면 13페이지 "권한 안내"를 참고하세요.

대시보드 Home

로그인 후 가장 먼저 보이는 화면 — 전체 퀘스트 운영 현황을 한눈에 확인합니다.

이 화면에서 확인할 수 있는 것

  • KPI 카드 6장 — 진행중/완료 퀘스트 수, 평균 진행률, 위험 퀘스트, 마감 임박, 활성 길드 수
  • Health(건강도) 분포 도넛 — 정상(초록)/정체(노랑)/위험(빨강) 비율과 위험 퀘스트 리스트
  • 이번 달 마감 예정 퀘스트 리스트
  • 에피소드별 진행률과 길드 랭킹 Top 순위
  • 최근 활동(Activity) 피드
헷갈리기 쉬운 점 — 위험/마감임박 리스트를 클릭하면 대시보드에 머무는 게 아니라 Quest 보드로 이동해 해당 퀘스트 상세가 열립니다. "여기서 바로 상세를 본다"고 생각하지 말고, 클릭하면 화면이 전환된다는 걸 기억하세요.

QUEST BOARD
06 / 20
Quest 보드 Quest > Board

전체 퀘스트를 표 형태로 확인하고, 새 퀘스트를 만드는 곳입니다.

할 수 있는 일

  • 퀘스트 목록 확인 — 상태·제목·유형·우선순위·진행률·마감일·파티장 등을 컬럼으로 표시
  • 그룹 보기(에피소드별 묶음) / 평면 보기 전환
  • 퀘스트 행을 클릭하면 우측에 상세 패널이 열림
  • 상단 "퀘스트 생성" 탭에서 새 퀘스트 등록 — 프로젝트/제목/파티장/마감일은 필수 입력
참고 — 뷰어(VIEWER) 역할은 퀘스트를 조회만 할 수 있고, 생성 버튼을 눌러도 서버에서 차단됩니다. 화면 폭이 좁아지면 유형·우선순위 등 일부 컬럼이 자동으로 숨겨져요.

QUEST CREATE
07 / 20
퀘스트 만들기 Quest > Board > 생성

Quest 보드 상단 "퀘스트 생성" 탭에서 새 퀘스트를 등록합니다.

조작 순서

  • 퀘스트 유형 선택 — 메인 퀘스트 / 길드 퀘스트 버튼 토글
  • 프로젝트 선택, 제목 입력 (둘 다 필수)
  • 파티장 입력 (필수) · 파티원은 쉼표(,)로 구분해 여러 명 입력 (선택)
  • 에피소드 선택 (선택) — 없으면 "없음 (독립 퀘스트)"
  • 시작일 / 마감일 입력 (마감일 필수)
  • 서브 퀘스트 입력 (선택) — "추가" 버튼으로 여러 개 등록 가능
  • "퀘스트 생성" 버튼 클릭
필수 입력 4가지 — 프로젝트 · 제목 · 파티장 · 마감일은 반드시 채워야 생성됩니다.
메인 vs 길드 퀘스트 — 메인 퀘스트는 조직(실·팀)의 성장 과제이고, 길드 퀘스트는 개인의 성장 과제입니다. 길드 파티원들이 스스로 목표나 과제를 만들어 진행하는 컨셉이에요. "길드 퀘스트"를 선택하면 길드 선택창이 나타나고, 길드를 고르면 파티장 이름이 해당 길드 리더로 자동 채워집니다. 완료 시 소속 멤버 전원에게 길드 포인트가 자동으로 부여돼요.
생성 직후 흐름 — 생성한 퀘스트는 바로 진행되는 것이 아니라 관리자 승인을 거친 뒤 진행이 시작됩니다.
팁 — 서브퀘스트는 생성 폼에서 미리 등록해도 되고, 나중에 상세 패널에서 추가해도 됩니다.

QUEST DETAIL
08 / 20
퀘스트 상세 관리 Quest Detail Panel

퀘스트 행을 클릭하면 열리는 상세 패널 — 진행률·서브퀘스트·스냅샷·액션 버튼을 관리하는 핵심 화면입니다.

위: 상세정보·진행율·액션 버튼 · 아래: 서브퀘스트 + 월간 스냅샷 섹션

진행율 업데이트

  • 서브퀘스트가 없으면 → "진행율 수정" 섹션에서 직접 수정, 이름과 진행 메모가 필수
  • 서브퀘스트가 있으면 → 메인 진행율은 서브퀘스트 평균으로 자동 계산되어 직접 수정이 잠깁니다
화면 안내 그대로 — "서브퀘스트가 있는 퀘스트입니다. 서브퀘스트는 상세 모달에서 관리합니다."

서브퀘스트

  • 진행 중(ACTIVE) 상태에서만 "+ 추가" 버튼 노출, 제목은 필수
  • 첫 서브퀘스트 추가 시 메인 진행율이 초기화된다는 확인창이 뜹니다
  • 개별 수정 — 진행율 슬라이더 / 진행 메모 / Blocker / 다음 액션 / 담당자

월간 스냅샷

  • 퀘스트의 월간 진행 기록 — 한 달에 1건만 생성 가능
  • "+ 생성" 클릭 → 확정 진행률 / 상태(정상·정체·위험) / 요약 / Blocker / 다음 계획 / 비고 입력
  • 자동 진행률은 생성 시점 값으로 고정되며, 당월에 이미 있으면 버튼이 비활성화됩니다
  • 작성 권한은 파티장 또는 매니저 이상만
중요 — 이 스냅샷이 월간 보고서의 원천 재료가 됩니다. 전월 스냅샷이 없으면 그 퀘스트는 보고서의 "전달比" 진척률 계산에서 제외돼요.

액션 버튼 4종

  • 완료 요청 (파티장만) · 파티 참여 · 연장 요청(새 마감일+사유) · 진행율 수정

퀘스트 라이프사이클 Status & Flow

퀘스트는 아래 9가지 상태 중 하나를 가지며, 정해진 순서로만 이동합니다.

상태한글 라벨게임 라벨의미
DRAFT초안Quest Draft생성됨, 아직 승인 전

COMPLETE_WAIT검증Clear Pending완료 요청 후 관리자 검증 대기
COMPLETED완료Quest Clear!완료 승인됨
FAILED실패Quest Failed마감 초과로 자동 실패
RETRY_WAIT재시도Retry Pending재도전 요청 후 승인 대기

생성(초안)→승인·진행→완료요청→검증→완료

EPISODE
10 / 20
Episode Quest > Episode

위: 에피소드 보드 · 아래: 에피소드 생성 폼

이 화면에서 할 수 있는 일

  • 에피소드 보드에서 진행 상황과 소속 퀘스트 확인
  • 에피소드 상태 확인 — 진행 중 / 검증 / 완료 / 보류 / 중단 (5종)
  • 에피소드 생성 탭 진입 (단, 생성 권한은 매니저 이상)

에피소드 만들기 — 필수 입력

  • 제목(60자 이내) / 마감일 / 우선순위 / 상태 / 담당자
  • "메인 퀘스트 연결" (선택) — 검색창으로 독립 퀘스트만 찾아 체크박스로 연결 가능
헷갈리기 쉬운 점 — "에피소드 생성" 버튼이 화면에 보이더라도, 일반 멤버 계정으로는 실제 생성이 되지 않습니다. 에피소드 생성·수정·삭제는 매니저(QA_MANAGER) 또는 관리자(ADMIN)만 가능해요.

MONITOR
11 / 20
Monitor Quest > Monitor

주의가 필요한 퀘스트만 모아서 보는 화면입니다.

이 화면에서 확인할 수 있는 것

  • 마감 임박 퀘스트 리스트 (D-N 배지)
  • 위험(danger) 퀘스트 — 블로커가 있고 오래 방치된 건
  • 실패(failed) 퀘스트 — 마감을 넘겨 자동 실패 처리된 건
  • 기간 탭(주간/월간/분기) 전환 및 항목 클릭 시 해당 퀘스트 상세로 이동
팁 — 팀 회의 전에 이 화면을 먼저 열어보면, 오늘 챙겨야 할 퀘스트를 빠르게 파악할 수 있어요.

Guild Hall & Guilds Guild Ranking

위: Guild Hall(랭킹 대시보드) · 아래: Guilds(길드 목록/창설)

이 화면에서 할 수 있는 일

  • Guild Hall — 길드 랭킹, 등급 분포, 성장 추이를 그래프로 확인
  • Guilds — 전체 길드 목록 조회, 길드 창설 탭 진입
  • 길드는 공통 관심 주제로 모인 개인 성장 파티 — 멤버의 퀘스트 완료 포인트가 쌓여 길드 등급이 올라갑니다
알아두면 좋은 점 — 한 사람이 소속될 수 있는 길드는 최대 3개까지입니다. Guild Hall은 독립적인 랭킹 대시보드로, Guilds 목록 화면과는 별개로 동작해요.

FAQ

FAQ / 팁 자주 헷갈리는 것들
완료 요청 버튼이 안 눌러져요.
진행률 100% 외에 진행 메모 작성이 필요합니다(서브퀘스트가 없는 경우). 서브퀘스트가 있다면 전부 완료 상태여야 해요.
에피소드를 만들려는데 반영이 안 돼요.
에피소드 생성은 매니저 이상만 가능합니다. 일반 멤버 계정에서는 서버가 거부합니다.
대시보드에서 위험 퀘스트를 클릭했더니 다른 화면으로 넘어가요.
정상 동작이에요. 대시보드는 요약만 보여주고, 상세 확인·조작은 Quest 보드에서 이루어집니다.
포인트를 얻었는데 이번 달 총합이 그대로예요.
월간 포인트 캡(25pt)에 도달했을 수 있어요. 초과분은 반영되지 않고 다음 달에 다시 쌓입니다.
진행율 수정 버튼이 안 먹혀요.
서브퀘스트가 1개 이상 있으면 메인 진행율은 서브퀘스트 평균으로 자동 집계되어 직접 수정이 잠깁니다. 개별 서브퀘스트를 수정해 주세요.
퀘스트 상태가 갑자기 "실패"로 바뀌었어요.
마감일이 지나면 시스템이 자동으로 진행 중 퀘스트를 실패로 전환합니다. 재도전 요청을 통해 다시 진행할 수 있어요.
반려된 퀘스트를 방치하면 어떻게 되나요?
반려 상태로 5일이 지나면 자동으로 "중단(DROP)"으로 전환됩니다. 빠르게 재제출하거나 확인이 필요해요.
회원 관리 탭이 안 보여요.
회원 관리는 관리자(ADMIN) 전용입니다. 매니저 계정에는 노출되지 않아요.
길드에 여러 개 가입할 수 있나요?
네, 최대 3개까지 동시에 소속될 수 있습니다.
스냅샷은 언제 써야 하나요?
월말에 한 번, 퀘스트당 월 1건 기록하는 걸 권장합니다. 이미 이번 달 스냅샷이 있으면 "생성" 버튼이 비활성화돼요.
보고서 코멘트가 입력 안 돼요.
확정(final)된 보고서는 코멘트가 잠깁니다. 또한 매니저 이상 + 초안(draft) 상태에서만 입력할 수 있어요.
