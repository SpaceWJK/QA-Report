확장성 분석

⚡ 수평 확장 — 프로젝트 수

사람: 프로젝트 추가 = 인력 추가 (선형 비용 증가).
AI: 프로젝트 추가 = 계정/디바이스 슬롯 추가 (한계비용 ≈ 인프라만).

1개 → 8개 확장 시 사람은 8배 인력이 필요하지만 AI는 스크립트 이식 + 계정 추가로 대응. 프로젝트별 TC 차이에 따른 시나리오 유지보수 비용만 발생.

⚡ 수직 확장 — 실행 빈도

사람: 3주 1회가 현실적 상한 (5인 운영 부담).
AI: 빌드 수급 시마다, 매일 밤, 회귀 의심 시마다 — 빈도 무제한.

"더 자주 테스트"는 결함의 조기 발견을 의미 — 결함 발견이 늦을수록 수정 비용은 기하급수로 증가하므로, 빈도 확장 자체가 간접 절감 효과를 만든다.

⚡ 시간 확장 — 야간/주말 활용

사람의 가용 시간: 주 40시간.
AI의 가용 시간: 주 168시간 (24/7).

야간 10시간만 활용해도 1계정당 매일 3회+ BAT 또는 Functional 1회+ 수행 가능. 비근무 시간대 활용은 사람으로는 원천 불가능한 영역.

⚡ 병렬 확장 — 계정/디바이스

현재 1계정 170분 — 직렬 실행 기준.
계정 N개 병렬 시 경과 시간 170/N분으로 단축 가능 (TC 분할 전제).

OS 4종(iOS/AOS/PC/화웨이) 병렬 인프라를 갖추면 사람 4인 병렬 구조를 그대로 기계로 복제 — 경과 시간 격차(6.8배)도 해소 가능.
