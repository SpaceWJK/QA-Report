확장성 분석

⚡ 수평 확장 — 프로젝트 수

사람: 프로젝트 추가 = 인력 추가 (선형 비용 증가).
AI: 프로젝트 추가 = 계정/디바이스 슬롯 추가 (한계비용 ≈ 인프라만).

1개 → 8개 확장 시 사람은 8배 인력이 필요하지만 AI는 스크립트 이식 + 계정 추가로 대응한다.
프로젝트별 TC 차이에 따른 시나리오 유지보수 비용만 발생한다.

⚡ 수직 확장 — 실행 빈도

사람: 3주 1회가 현실적 상한 (5인 운영 부담).
AI: 빌드 수급 시마다, 매일 밤, 회귀 의심 시마다 — 빈도 무제한.

"더 자주 테스트"는 결함의 조기 발견을 의미한다.
결함 발견이 늦을수록 수정 비용은 기하급수로 증가하므로, 빈도 확장 자체가 간접 절감 효과를 만든다.

⚡ 시간 확장 — 야간/주말 활용

사람의 가용 시간: 주 40시간.
AI의 가용 시간: 주 168시간 (24/7).

야간 10시간만 활용해도 무인 BAT/Functional 실행이 가능하다.
비근무 시간대 활용은 사람으로는 원천 불가능한 영역이다.

⚡ 병렬 확장 — 계정/디바이스

현재 1계정 직렬 수행: 4 OS 총 680분.
계정 N개 병렬 시 경과 시간은 680/N분으로 단축된다 (TC 분할 전제).

사람 4인 병렬 구조를 그대로 기계로 복제하고, 그 이상으로 확장할 수 있다.
상세 잠재력은 아래 하이라이트 참조.

🚀 병렬 확장 잠재력 — AI 자동화의 진짜 상한은 계정 수가 결정한다

현재 측정치(680분)는 1계정 직렬 실행이라는 가장 보수적인 조건의 숫자다.
사람의 병렬화는 인원 수에 비례해 인건비가 증가하지만, AI의 병렬화는 계정과 디바이스 슬롯만 추가하면 된다.
환경만 갖춰지면 10계정, 20계정으로 확장 가능하며, 테스트 수행 시간은 그에 반비례해 기하급수적으로 단축된다.

등급항목내용완화 방안
HIGH 커버리지 상한 (현재 60%) 총 TC의 40%는 AI 수행 불가 영역 — 사람 테스트가 구조적으로 필수.
인원 "0명화"는 불가능하며, 절감은 부분 대체에 그침.		 불가 영역 TC를 유형 분류 → 자동화 가능/불가 경계를 명시하고 분기별 재평가
MED 검수 오버헤드 AI 결과의 사람 검수가 절감분을 잠식 (25분 → 실질 15분, 40% 손실).
신뢰도가 낮으면 검수 비중이 더 커질 수 있음.		 오탐/미탐률 측정 → 신뢰 구간 확보 시 샘플링 검수로 전환
MED 시나리오 유지보수 비용 8개 프로젝트 확장 시 UI 변경·패치마다 자동화 스크립트 수정 필요.
이 비용은 현재 측정에 미포함.		 프로젝트당 유지보수 시간 실측 → 절감분에서 차감한 순절감 재산출
LOW 경과 시간 (1계정 680분) 긴급 빌드 검증 시 사람 4인 병렬(25분)보다 27배 느림 — 당일 출시 등 긴급 상황에서는 사람이 우위. 긴급=사람, 정기/야간=AI 역할 분담 + 계정 병렬화(N계정 → 680/N분)로 단축
LOW Functional 미지원 현재 가장 방대한 테스트(3h+)는 AI 수행 불가 — 최대 절감 레버가 아직 잠겨 있음. 로드맵 Phase 4 — BAT 안정화 후 Functional TC 단계적 확장
▼ 핵심 제약 — 인원 절감의 비선형성 AI가 60%를 대체해도 잔여 40% TC는 여전히 OS 4종 병렬 인력을 요구한다.
즉 "60% 절감 = 인원 60% 감축"이 아니다 — 테스트 인원은 단계적으로만 줄어든다.
절감 효과는 당분간 "인원 감축"이 아니라 "동일 인원의 처리량 확대"(더 많은 프로젝트·더 자주)로 실현하는 것이 현실적이다.

정량 외 효과 (정성)

🛡 빌드 안정성

에픽세븐도 빌드 수급 시마다 반복 BAT 실행이 가능해진다.
추가 인력 0으로 회귀 결함의 조기 발견 창구가 생긴다.
결함 수정 비용은 발견 시점이 늦을수록 증가하므로, 직접 절감으로 환산되지 않는 숨은 가치다.

🤝 개발사 측 절감

개발사 사용성이 개선되면 개발사 측 자체 테스트 리소스 절감을 기대할 수 있다.
절감 효과가 퍼블리셔 내부를 넘어 파트너까지 확장되는 구조다.
협업 관계 가치로도 작용한다.

📊 측정 가능성

AI 테스트는 실행 로그·결과가 구조화되어 품질 지표의 시계열 축적이 자동으로 이뤄진다.
사람 테스트 대비 추적성(traceability)·재현성이 높다.
품질 리포팅 비용도 함께 절감된다.
