  • 기존 서비스 개선 (약 30%) — QA-Dashboard, QA-Workflow, Slack Bot(QA-Supporter) 등 운영 서비스의 기능 개선·안정화.

  • 운영 서비스 개선 배포 다수

5. 7월 사용 상세 — TC 자동 생성 시스템

기획 리뷰·TC 자동 생성 시스템 + QA-Workflow 통합 — 약 80% 제공 기타 서비스 기능 개선 — 약 20%
  • TC 자동 생성 시스템·엔진 개발 (약 80%) — 기획서를 AI가 리뷰하고 테스트케이스(TC)를 자동 생성하는 엔진 개발, 그리고 이를 QA-Workflow에 통합. QA 업무를 '관리'하는 시스템에서 '수행'까지 하는 시스템으로 진화시키는 전략 과제.
  • 기타 서비스 개선 (약 20%) — 운영 서비스 유지·개선.

7월 산출물

  • TC 자동 생성 엔진 (게임별 학습 구조 — 프로젝트마다 분류 규칙을 학습해 정확도 향상)
  • QA-Workflow 내 TC Viewer / Testing 기능 통합
  • 야간 자동 배치 (무인 TC 생성 파이프라인)
