게임 QA 사이클 전체를 담은 도메인 특화 업무 플랫폼 — 설계 기술서

일감 트래커가 아니다. JIRA·엑셀·슬랙으로 분산된 운영을 단일 실시간 인터페이스로 통합한다. 빌드 사이클 전주기(수급 → 테스트 → 배포 → 회고) 관리, 9가지 테스트 유형 개별 트래킹, 기한/진척도/빌드 상태 기반 자동 리스크 계산, 실시간 팀 상태 가시화 + 근태 연동, 타겟별 기록 카드 + 트렌드 분석을 제공한다.

01 · Domain · Problem · Philosophy

개요

게임 QA 도메인의 특수성 — 빌드 차수별 수급, 9종 테스트 개별 진척, 라이브 이슈↔업데이트 타겟 매핑, 팀원 상태 실시간 가시화, 근태·타이머 연동.

해결하는 문제 5가지
문제해결
분산 운영JIRA + 엑셀 + 슬랙 → 단일 통합 인터페이스
수동 리스크기한/진척도/빌드 상태 기반 자동 계산
정적 팀 상태픽셀 오피스 + 실시간 소켓
텍스트 공지 회의픽셀 오피스 내 채팅 회의실
분산 근태일감 타이머와 자동 연동
핵심 설계 원칙 3가지
01 · Build Cycle
빌드 사이클 = 회고 단위
UPDATE 타겟 1건 = 기록 카드 1장. 빌드 사이클을 회고의 기본 단위로 삼는다.
02 · Stats Anchor
Stats 위치 고정
이슈/커뮤니티 stats는 활성 UPDATE 빌드 ID에 누적. 지표의 귀속 위치를 고정한다.
03 · Real-Time Sync
실시간 동기화
REST 읽기/쓰기 + Socket.io로 전 세션 동시 갱신. 한 사람의 변경이 모두에게 즉시 반영.
02 · 11 Sidebar Menus

핵심 기능

11개 사이드바 메뉴가 게임 QA 사이클 전 단계를 커버한다. 각 메뉴는 도메인 로직 + 실시간 동기화를 공유한다.

01 · Dashboard 📊
관제 대시보드
KPI 5 + 위험 일감 테이블 + 오늘 일정 + 팀원 상태 요약. 프로젝트 건강도(지연 비율 + 위험 max) + 9가지 팀 상태 실시간 카운트.
02 · Tasks 📋
9종 테스트 트래킹
자동 리스크 4단계(CRITICAL~LOW) · Timer + 수동 시간 추가(최대 24h/회) · 일감-빌드 연동 · 의존성(BLOCKS) · Optimistic Locking.

03 · Builds 🏗️
6단계 빌드 FSM
Received → Testing → Completion Pending → Test Done → Approved → Released. APPROVED→RECEIVED 회수 허용. iOS/AOS/PC × APP/CDN/SERVER 정렬, 반려 이력 JSON 추적.
04 · Team 👥
팀 현황 + 부하
팀원 현황·업무 부하·프로필·실시간 상태 동기화. 역할/상태/팀별 필터, 개인별 할당 일감 + 경고 임계값. teamStatus 변경 전 세션 브로드캐스트.
05 · Pixel Office 🎮💬
픽셀 오피스 + 회의
18석 그리드 + 실시간 채팅 회의실. 근태 연동(오늘 출근자 자리 배정), 공개/비공개 방 + ADMIN 감독 모드. 참여 시 IN_MEETING 자동, 전원 퇴장 시 자동 닫힘.
06 · Calendar 📅
통합 캘린더
월간 그리드 + 간트 + 빌드 타임라인 + Today Indicator. 프로젝트 일감 + 개인 일정 통합. 휴가/출장/반차/재택/외근/회의 + 공휴일 자동.
07 · Attendance ⏰
근태 관리
출퇴근 클록 + 일일 기록. 월간 누적(정상/지각/조퇴/결근). 기간별·프로젝트별 인원/시간 리포트.
08 · QA Tool 🛠️
게임별 운영 도구
EP7: 서버 시간 변경/원복·PHPSESSID·전월드 원복. CZN: Jenkins(배포/CDN/버전)·Preflight LRU. LDN/LNA: 기본 도구. 공용 즐겨찾기(레드마인/가이드/운영툴).
09 · Issues 🔍

10 · History 📜

06 · Request · Validation · Broadcast

데이터 파이프라인

일감 상태 전이 + 실시간 브로드캐스트. 사용자 액션은 3중 게이트를 통과한 뒤 전 세션에 방송된다.

상태 변경 예시 · 일감 전이 (User A → User B 동시 감지)
STEP 01User A 상태 변경 요청

Hook · TanStack에서 IN_PROGRESS 전이 트리거.

STEP 02Request

PUT /api/tasks/:id 전송.

STEP 03Middleware

JWT + Zod 통과.

STEP 04Service

FSM · 버전 체크 · UPDATE 실행. 3중 게이트가 여기서 작동.

STEP 05Response

{ok, task, version:N+1} 반환.

STEP 06Broadcast → User B

io.to('global').emit('task:statusChanged') → User B의 invalidateQueries(['tasks']) 자동 갱신 + UI refresh.

3중 게이트 검증
Gate 1 · FSM
상태 전이 검증
VALID_TRANSITIONS의 허용 전이만 통과. Task 9상태 · Build 6단계 · 회의실 상태를 서버에서 엄격 검증.
Gate 2 · Optimistic Lock
동시 편집 방지
version 필드로 conflict 감지. A·B가 같은 일감 동시 수정 시 후착 트랜잭션 자동 거부.
Gate 3 · ActivityLog
감사 기록
모든 상태 변경 기록 — 누가·언제·무엇을·왜. 회고/감사/보안 점검용.
자동 리스크 계산 (risk.service.ts)
# 기대 진척도 expected = (today - startDate) / (dueDate - startDate) × 100 # 실제 진척도 total = progressTotal 차이 = expected - total IF 차이 > 40% && 기한 임박 → CRITICAL ELIF 차이 > 25% → HIGH ELIF 차이 > 10% → MEDIUM ELSE → LOW
Appendix · Glossary

용어 사전

빌드 사이클·테스트 유형·실시간 동기화 핵심 용어.

용어설명
FSMFinite State Machine. 빌드/일감/회의실 상태 전이 검증.
Optimistic Lockversion 필드로 동시 수정 conflict 감지.
BVTBuild Verification Test — 빌드 수급 직후 최소 동작 확인.
BATBuild Acceptance Test — 상세 검증 전 단계 수용 여부 판정.
UPDATE 타겟정규 업데이트 배포 기준일. 기록 카드 1장 = UPDATE 타겟 1건.
HOTFIX라이브 긴급 패치. 현재 UPDATE 윈도우에 귀속되어 집계.
ROI v0.2Benefit(이슈 처리 비용) / Cost(TaskTimeLog × 시급) × 100.
teamStatus팀원 실시간 상태 (AVAILABLE / IN_MEETING / OFF_WORK 등 9종).
TanStack Query서버 state 캐싱 + invalidation 라이브러리 (React Query).
MCP HTTPStreamable HTTP 기반 MCP 프로토콜. 사내 JIRA/Wiki 접근.
pm2Node.js 프로세스 매니저. ecosystem.config.cjs로 다중 앱 관리.
PrismaType-safe ORM. 스키마 기반 migration 관리.
Tier A / Tier BPBI 엑셀 export 계층 (A: 38컬럼 / B: 51컬럼 확장 메트릭).
