06 · Aggregation

집계 · KPI 계산식

버그를 비율 모수에서 분리하고, game_sentiment를 우선한다. 비율 모수는 비버그 게시글, 버그는 별도 카운트.

# voice_aggregator.precompute_kpi() — 집계 규칙 eff_sentiment = COALESCE(game_sentiment, sentiment_class) # 게임 감성 우선 bug_flag=1 → bug 카운트만 (긍/부/중립 비율 제외) tier2 → uncertain_count 추가 (비율엔 포함) tier3 → total_posts 미포함 (tier_label IS NOT NULL 필터) none/NULL → 중립(neu) 편입 # 프론트 비율 (VoiceKpiCards.tsx:54) — 비버그 모수 nonBug = total_posts - bug_count posRate = pos_count / nonBug × 100 negRate = neg_count / nonBug × 100 neuRate = neu_count / nonBug × 100 # 전기간 델타 delta = 현재기간 비율 - resolvePrevPeriod(동일길이 직전) 비율 (%p)
TOP5 토픽 (voice_aggregator.py:544): Haiku가 최근 30일 샘플(≤60건)에서 구체 토픽 5개 추출 → search_query LIKE COUNT로 post_count 보정(환각 제거) → 0건 제거 → 내림차순. 캐시 24h. KPI snapshot TTL 5m / 카테고리 24h / 토픽요약 7일.

10 · Topic Summary + Citation

토픽 요약 · 인용 생성

토픽 샘플을 Haiku로 bullet 5개 요약하고, 각 bullet에 원문 게시글 인용번호를 부착·검증한다. signature 기반 캐시로 재생성을 억제한다.

01샘플 수집

fetch_topic_samples() ≤50건, ko_translation 우선

02signature

SHA256(sorted post_ids + classified_at)[:16] — 캐시 무효화 키

03캐시 판정

signature 일치 AND 7일 이내 → 캐시 hit. miss → Haiku 호출

04프롬프트

"[N] ko_title — body[:200]" 나열 → bullet 5 + 각 bullet [N] 인용 → JSON {bullets:[{text, citations:[]}]}

05검증·저장

1≤c≤len(samples) 범위 외 인용 제거(CRITICAL-F1), bullet≤5·text≤200자. upsert 캐시.

LLM 직접 토픽 추출(generate_llm_topics): Haiku가 60건 샘플에서 구체 토픽(캐릭터/스킬명 포함) 5개 추출 → _enrich_topic_counts가 search_query LIKE COUNT로 실제 건수 보정(환각 제거). 카테고리 요약은 별도 24h 캐시.
