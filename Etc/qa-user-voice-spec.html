유저 반응 동향 — 통합 QA-Dashboard의 USER 탭

"유저 동향 분석 시스템"(비전 문서 시스템 2)을 실제 운영 기능으로 구현해 통합 QA-Dashboard에 USER 탭으로 착지시킨 것. 게임 커뮤니티 게시글을 24/7 수집 → NLP 감성·버그 분석 → 토픽 클러스터링하여, 유저가 지금 무엇에 반응하는지를 긍정/부정/중립 비율 · 버그 의심 · TOP 토픽 · 인용 기반 요약으로 보여준다. JIRA·QA·REPORT·OPS·DEVICE·QUEST와 동일한 단일 인터페이스의 한 탭.

01 · Purpose & Value

목적과 가치

"버그 검출 사후 대응"에서 "유저 반응 예측적 파악"으로. 흩어진 커뮤니티 여론을 운영 가능한 지표로 변환한다.

Why · 문제
여론은 흩어져 있다
유저 반응이 다수 커뮤니티에 분산. 수동으로는 "지금 무엇이 문제인지" 적시 포착 불가. 버그 신호도 게시글 속에 묻힌다.
How · 방식
수집 → NLP → 토픽화
24/7 크롤링 → 감성 3분류 + 버그 키워드 추출 → 토픽 클러스터링 → bullet 요약 + 원문 인용. 사람이 읽기 전에 구조화한다.
Value · 가치
예측적 VOC 관리
감성 추세·버그 의심 조기 탐지로 대응 시간 단축. 밸런스/신캐/메타 논쟁을 데이터로 포착해 운영·기획 의사결정 지원.
전략적 의미: 비전 문서의 4대 시스템 중 시스템 2(유저 동향 분석)를 가장 먼저 실물로 구현한 사례. 추상적 비전이 통합 대시보드의 동작하는 탭으로 착지했음을 증명한다.

Appendix · Glossary

용어 사전

USER 탭 핵심 용어.

용어설명
User Voice (USER 탭)통합 QA-Dashboard에서 유저 커뮤니티 반응을 NLP로 분석·표출하는 탭.
감성 3분류게시글을 긍정/부정/중립으로 분류해 비율·건수로 집계.
버그 의심NLP가 버그 관련 키워드로 분류한 게시글 수. 버그 제보 큐로 연결.
토픽 클러스터유사 게시글을 묶은 주제 단위. TOP5를 게시글 수 기준으로 노출.
bullet 요약 + 인용토픽별 핵심 쟁점 5개 + 각 항목 근거 게시글 번호([n]).
전주/전월 델타주기 기준 직전 구간 대비 증감(%p / 건). 추세 파악용.
voice-worker커뮤니티 수집·NLP 분석을 주기 실행하는 백그라운드 워커.
NLP 지연 경고분석 지연이 임계 초과 시 데이터 비최신 상태를 알리는 상단 배너.
